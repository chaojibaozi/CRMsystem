# 任务摘要：CRM 系统字体大小优化建议

## 目标
为 CRMSystem (index.html) 提供字体大小优化方案，生成可直接操作的文档。

## 工作内容
1. 全面分析 index.html 中所有 font-size 使用情况（共扫描到 20+ 个离散字号值）
2. 诊断出三大核心问题：层级混乱、10-12px 小字泛滥、表格层级扁平
3. 设计 8 级字号体系 CSS 变量方案
4. 逐模块输出改前/改后代码（14 个模块：表格、sidebar、topbar、H2、浮层、BI KPI、Badge、错误提示、Modal、时间线、Auth、宠物猫等）
5. 按 P0~P3 排好优先级和修改清单
6. 输出 .md 和 .txt 两个版本，方便 Trae 直接使用

## 输出文件
- `CRMSystem-字体大小优化建议_2026-06-24.md` — 详细文档
- `CRMSystem-字体大小优化建议_2026-06-24.txt` — 纯文本版，供 Trae 优化
