# 2026-06-19 工作记录

## 客户等级与标签功能调整

### 背景
客户可能有多个待合作业务（多渠道），因此全局客户等级不合理，等级应在渠道内使用。

### 修改内容

#### 1. 客户列表模块移除等级（index.html）
- 移除主列表筛选栏的「客户等级」下拉筛选器
- 移除主列表表头的等级列（`<th data-sort="level">等级 ▾</th>`）
- 移除 `applyListFilters()` 中的等级过滤逻辑
- 移除筛选按钮事件绑定中的等级取值和重置代码
- 移除 `renderFilterLevels()` 函数的两次调用（主列表渲染时不再渲染等级筛选）

#### 2. 自定义列配置添加客户标签（index.html）
- 在 `getColumnConfig()` 中注入 `{ key: 'tags', visible: true }` 列，使主列表可在「自定义标签」弹窗中切换标签列的显示/隐藏

#### 3. 新增/编辑客户表单移除等级（index.html）
- 移除表单业务信息区块中的「客户等级（单选）」UI（包含 `level-radio` 胶囊选择器和 `level-hidden` 隐藏输入）
- 移除等级点击事件处理代码
- 移除重置表单中的等级还原代码
- 移除数据收集中 `level` 的渠道字段特殊处理
- 移除保存时 `setChannelField(old, _currentChannel, 'level', ...)` 代码

### 保留
- 渠道列表（GEO/360/Bing/百度/抖音/闲鱼/作废）的等级筛选器保留，等级在渠道内仍可使用
- 客户详情页的等级展示保留
- BI 看板的等级图表保留
- 设置中心的等级管理功能保留
- `getChannelColumnConfig()` 中 level 列的注入保留
