## BI 看板 Bug 修复

检查并修复了 `index.html` 单文件 CRM 系统中 BI 看板模块的以下 Bug：

1. **[严重] Bug #1 - todayStart 作用域错误**：`renderBIAnalysisGrid` 函数内 IIFE 引用了 `todayStart`，但该变量是 `renderBIPage` 的局部变量，函数签名未传入，导致 ReferenceError。修复：在调用处和函数签名都加入 `todayStart` 参数。

2. **[严重] Bug #2 - 标签排行永不渲染**：`renderAllBICharts` 使用字符串拼接约定（`draw` + name + `Chart`）调用图表函数，但标签排行函数名为 `drawBITagRank`（无 `Chart` 后缀），导致调用失败。修复：改为直接引用函数对象的对象数组，消除命名约定依赖。

3. **[中] Bug #3 - 业绩缺口预警逻辑不合理**：原来以 `total * 15%` 为期望值，导致每月几乎都触发预警。修复：改为以本月新增客户数的 30% 作为期望成交量。

4. **[中] Bug #4 - 跟进时效图包含非活跃客户**：时效分析错误统计已成交/流失客户，数据失真。修复：只统计 status 非 `已成交`/`流失` 的活跃客户，并改用创建时间兜底（而非固定归入 >1月 组）。

5. **[中] Bug #5 - 性别环形图 rightX 坐标错误**：`W/2 + halfW/2` 导致圆心偏出画布右侧被裁切。修复：以右半区域的实际中心计算 rightX。

6. **[轻] Bug #6 - 预算分布图 rightX 与左侧 halfW 分割逻辑不一致**：布局错位。修复：统一用 `padding.left + halfW + 40` 为右区起点。

7. **[轻] Bug #7 - exportBIReportCSV 文件格式问题**：文件名 `.csv` 但内容是纯文本，且无 BOM 导致中文乱码。修复：改为导出带 UTF-8 BOM 的 `.txt` 文件，使用 Blob URL 下载。

8. **[轻] Bug #8 - churnList DOM 判空缺失**：修复流失预判区块 innerHTML 写入前未检查 DOM 存在性。

9. **[轻] Bug #9 - 深色模式环形图内圈颜色读白**：`getComputedStyle(document.body)` 读 CSS 变量可能为空，兜底 `#fff` 在深色主题下导致内圈发白。修复：改从 documentElement 读取变量，并检测 data-theme 属性决定文字颜色。

10. **[优化] 预计本月新增算法**：原 `monthNew * 1.3` 无意义，改为按已过天数线性外推到月末。

## 全局猫咪小宠物浮层

新增全局猫咪浮层（IIFE 模块，`window.CRMPet` 暴露调试接口），包含：

- **提醒逻辑**：扫描 `Store.data.customers` 中超过 3 天/7 天未跟进客户，7天以上为紧急（红点+快速脉冲），3-7天为一般提醒（黄点）。
- **气泡提醒**：带标题、正文、操作按钮（立即查看/知道了/稍后提醒），稍后提醒延迟 1 小时重新检查。

- **拖拽**：支持 mouse + touch 拖拽定位，边界自适应。
- **定时调度**：启动后 3 秒首次扫描，之后每 30 分钟自动检查；无操作 5 分钟后进入睡眠状态。

### 猫咪扫描时序根本修复
- 根本原因：`scheduleCheck()` 的轮询在 IIFE 启动时立即运行，此时 `Store.data.customers` 可能是空数组（已通过 `[]` 初始化），轮询误判"已就绪"而 `clearInterval` 退出，解锁后真实数据加载完毕时无人再触发扫描。
- 修复：在 `enterApp()` 解锁成功后（`navigate('dashboard')` 之后）直接调用 `window.CRMPet.check()`，延迟 1200ms 等渲染完成。
- `scheduleCheck()` 移除首次扫描轮询，只保留 30 分钟定期复查和 5 分钟睡眠检测。

### 仪表盘预警模块加载异常修复
- Bug #13：`renderAlertSection` 行3791，overdueList `.map()` 解构赋值语法错误 — `{ ...c, _isOverdue, _overdueDays: overdueDays, _severity, _severityLabel, _reason }` 中 `_isOverdue`、`_severity`、`_severityLabel`、`_reason` 缺少属性值（ES6 shorthand property 要求变量名和属性名相同，但 `isOverdue`/`severity`/`severityLabel`/`reason` 与目标属性名不一致），导致 `ReferenceError` 或赋值错误。修复：改为 `Object.assign({}, c, { _isOverdue: isOverdue, ... })`。
- Bug #14：行3811，upcomingList 同样问题 — `{ ...c, _isToday, _daysUntil, _label, _cssClass }`，同样改为 `Object.assign`。
- Bug #15：行3977，`Store.settings.followupDays` 无短路保护，与行3767 不一致。修复：`(Store.settings && Store.settings.followupDays) || 3`。
- 注：Trae 同步修改了该文件新增预警模块，但 ES6 解构语法写法有误导致整个 `renderAlertSection` 在 `try` 块中抛异常，被 `catch` 捕获后显示"预警模块加载异常"。
