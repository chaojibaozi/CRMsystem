## 自动保存机制实现（3 秒间隔）

为系统新增全局自动保存机制，防止意外关闭导致数据丢失：

1. **Store 对象新增字段**：
   - `_dirty: false` — 脏数据标记，有未保存变更时为 true
   - `markDirty()` — 标记数据已变更（供自动保存轮询使用）
   - `save()` 保存成功后自动清除 `_dirty = false`

2. **自动保存定时器**：
   - `autoSaveTimer` + `startAutoSave()` / `stopAutoSave()` 控制生命周期
   - `AUTO_SAVE_INTERVAL_MS = 3000`，每 3 秒检查 `Store.unlocked` 状态，若已解锁则调用 `Store.save()`
   - 无条件保存（不依赖 dirty 标记），确保任何数据变更都能被覆盖
   - 保存失败仅 `console.warn`，不中断用户操作

3. **集成点**：
   - `enterApp()` — 解锁后调用 `startAutoSave()` 启动自动保存
   - `resetIdleTimer()` idle logout 回调 — 登出前最后保存一次（如果 dirty），然后调用 `stopAutoSave()`
   - 忘记密码重置 — 调用 `stopAutoSave()` 并重置 `_dirty = false`

4. **JavaScript 语法验证通过**（Node.js `new Function()` 检查）。
