## 忘记密码按钮修复

**问题**：登录页"忘记密码"按钮在设置了密保问题后显示"通过密保问题重置密码"，但点击后读取 `Store.data.securityQuestions` 失败（Store 在登录页未解密），永远报"未设置密保问题"。

**根因**：密保问题存储在加密主数据中（需密码解密），但用户正是忘记密码才想通过密保重置——死循环。

**修复**：
- 新增 `SQ_STORAGE_KEY = 'crm_sq_data_v1'` 独立 localStorage key
- 保存/清除密保时同步写入/清除独立存储
- `enterApp()` 登录成功后同步加密数据到独立存储
- `openResetBySecurityQuestionsDialog` 改为从独立存储读取
- `initSecurityQuestionsUI()` 同步时也同步独立存储

## 密码恢复保险箱（解决重置密码时的数据丢失问题）

**问题**：通过密保问题重置密码会导致所有加密数据丢失（旧密码不可知）。

**方案**：新增「密码恢复保险箱」机制。在密保问题首次设置时，用密保答案派生 PBKDF2 密钥，将当前密码加密后独立存储。重置时：
1. 用户回答密保问题 → 验证通过
2. 读取保险箱，用答案派生密钥解密 → 获得旧密码
3. 用旧密码解密主数据 → 用新密码重新加密保存 → 更新保险箱

**涉及修改点**：
- 新增 `SQ_PW_VAULT_KEY` 常量
- `btnSqSave`：设置密保时同步创建保险箱
- `btnSqClear`：清除密保时同步清除保险箱
- `btn-change-password`：改密码时更新保险箱
- `enterApp()`：登录时若保险箱缺失自动重建
- `openSetNewPasswordDialog`：重置时优先尝试保险箱恢复旧数据，兜底回退为空数据重置（同时清除残留 SQ 数据）
- `Store.data` 默认值和所有重置路径统一补齐 `securityQuestions` 和 `poolCustomers` 字段

## 密保保存按钮无响应修复

**问题**：`btnSqSave.onclick` 中 `Store.save().then(...)` 缺少 `.catch()`，若保存因竞态或加密异常 reject，用户收不到任何反馈。

**修复**：
- 改为 `async` 函数 + `try-catch` 包裹整个 handler
- `Store.save()` 改为 `await Store.save()` + 内层 try-catch，失败时显示错误 toast
- DOM 读取改用 `?.` 可选链防空
- `btnSqClear` 同步补上 `.catch()` 错误提示
